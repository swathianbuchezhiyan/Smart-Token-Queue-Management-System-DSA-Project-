Smart Token Queue Management System
Project Overview
This project is a Python based Smart Token Queue Management System that simulates real world hospital or bank queue operations. It uses Queue and Priority Queue concepts to manage customers efficiently using token numbers and priority levels.
Features
Token generation for each patient or customer
Priority based scheduling for Emergency and Normal cases
FIFO queue management
Interactive menu driven system
Real time queue display
Serve patients based on priority
Technologies Used
Python
Data Structures and Algorithms
Queue
Sorting and Priority Logic
How It Works
User selects an option from menu
Patient is added with a token number
Priority is assigned (0 for Emergency, 1 for Normal)
System sorts and serves based on priority
Queue updates after each operation
Example Output
Token 1 assigned to Patient A
Token 2 assigned to Patient B
Serving Token 2 Emergency
Remaining queue displayed
