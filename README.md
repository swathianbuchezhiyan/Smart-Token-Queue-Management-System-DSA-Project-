Smart Token Queue Management System (DSA Project)
Project Overview
This project is a Python based Smart Token Queue Management System that simulates real world hospital or bank queue handling using Data Structures and Algorithms. It manages customers using token numbers and priority based scheduling.
Objective
To simulate a real time queue management system using Queue and Priority Queue concepts and provide an interactive menu driven application.
Features
Token generation for each patient
Priority based scheduling (Emergency and Normal)
FIFO queue system
Interactive menu system
Real time queue display
Serve patients based on priority
Technologies Used
Python
Data Structures (Queue)
Basic Algorithms (Sorting and Priority handling)
Project Workflow
1. Add Patient
User enters name and priority
System assigns token number
2. Show Queue
Displays all waiting patients
Shows priority status
3. Serve Patient
Emergency patients served first
Queue updated after serving
4. Exit
Stops the system
Example Output
Token 1 assigned to Patient A
Token 2 assigned to Patient B
Serving Token 2 Emergency
Current Queue displayed
Learning Outcomes
Understanding Queue data structure
Priority scheduling concept
Real world system simulation
Interactive Python programming
